# Runbook

这是一页日常运行协议。只有卡住、验证、维护时才需要看。

## 0. 默认原则

先学习，再维护。文档服务学习，不反过来支配学习。

## 1. 快速学习模式

适合：一个明确概念、一个报错、一个小问题。

```text
当前输入
-> AI 先判断能力节点和台阶
-> 给最小解释
-> 给 1 个小练习
-> 给 1 个无 AI 验证任务
```

默认不写文档。除非验证结果暴露前置缺口或夹生风险。

## 2. 深挖模式

适合：基础概念不清、越学越乱、同类问题反复失败。

```text
定位问题场景
-> 找能力链
-> 找最近前置节点
-> 拆成半步台阶
-> 学习后立即做检索/反例/变式
-> 无 AI 验证
```

如果失败，先回退前置，不继续扩展资料。

## 3. 验证模式

适合：感觉自己懂了，准备确认能力。

验证必须满足至少一项：

- 无 AI 解释一个概念的边界和反例
- 无 AI 完成一个变式任务
- 无 AI 排查一个最小问题
- 间隔一段时间后仍能复现关键推导

通过后写 `docs/evidence/`；只有稳定证据才建议写 `docs/baseline/confirmed.md`。

## 4. 维护模式

适合：学习结果值得留下，或 map 关系需要修正。

维护顺序：

```text
evidence -> map -> current -> baseline
```

不要反过来先写 baseline。baseline 是最后收敛，不是开头愿望。

## 5. AI 每次维护后必须汇报

```text
新增了什么：
修改了什么：
删除了什么：
哪些是候选判断：
哪些需要用户确认：
哪些没有写入 baseline：
```

## 6. 停止条件

出现下面任一情况，停止扩展，回到当前台阶：

- 新资料越来越多，但验证任务没有变清楚
- AI 解释越来越长，但自己复述不了
- 发现前置概念不稳
- 当前任务已经能验证，不需要继续查资料
