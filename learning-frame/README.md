# Learning Frame：面向个人能力形成的低摩擦学习骨架

`learning-frame` 是给个人长期学习使用的 AI 约束框架。它不是课程表、资料库、笔记法或通用学习方法论；它的目标很具体：

> 让学习围绕可验证的能力证据持续收敛，帮助我快速学习，同时尽量避免“看懂了但不会”“会用但说不清”“靠 AI 才能完成”的夹生饭。

## 核心定位

- 强骨架，弱内部结构
- 内容和能力证据优先，不追求模板外观完整
- AI 负责结构维护和诊断，人负责掌握状态裁决
- 学习时可以用 AI，验证时必须能脱离 AI

## 从 project-frame 转换过来的思想

`project-frame` 的吸引子是项目真相和交付收敛；`learning-frame` 的吸引子是能力证据和路线修正。

| 项目框架 | 学习框架 |
|---|---|
| 收敛到项目当前真相 | 收敛到已验证能力真相 |
| 交付物是否完成 | 能力是否可复现、可解释、可迁移 |
| live repo 决定实际行为 | 无 AI 验证决定掌握状态 |
| baseline 记录项目认可结论 | baseline 记录已确认能力事实 |
| plans/logs 记录任务轨迹 | current/evidence 记录学习台阶与验证证据 |

## 固定层

- `docs/current/`
  - 当前正在补的一个能力台阶。
- `docs/map/`
  - 问题场景、能力链、能力节点构成的能力图。
- `docs/evidence/`
  - 无 AI 测试、迁移任务、复盘结果等掌握证据。
- `docs/baseline/`
  - 已确认能力真相、长期收敛方向、个人学习原则。
- `docs/input/`
  - 本轮原始材料、问题、截图、对话、临时输入。
- `docs/references/`
  - 长期可引用资料、论文、教程、外部文章。

## 最小学习闭环

```text
真实问题/学习目标
-> 能力地图定位
-> 当前台阶
-> AI 辅助理解与练习
-> 无 AI 验证
-> evidence 记录
-> 必要时修正 map / baseline
```

## 最硬边界

- 没有无 AI 验证，不得把能力标为 confirmed。
- AI 可以维护结构，但不能静默改变主学习路线。
- `references/` 不是已掌握事实。
- `input/` 不是结论区。
- `baseline/` 不做资料堆，只放已确认能力真相和长期原则。
- 小学习、小问答、小练习默认不写 evidence；只有验证结果或路线修正值得保留时再维护。
