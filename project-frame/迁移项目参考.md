# SVN 项目内使用 Git 管理项目知识库

## 目标

代码仍由 SVN 管理。
本机 Git 只管理：

```text
.gitignore
AGENTS.md
START-HERE-after-copy.md
docs/**
tools/**
```

---

## 1. 进入 SVN 项目根目录

```bash
cd /path/to/svn-project
```

---

## 2. 初始化 Git

```bash
git init
```

---

## 3. 创建 `.gitignore`

```bash
cat > .gitignore <<'EOF'
# 默认忽略 SVN 项目里的所有内容
*

# Git 自己需要管理的规则文件
!.gitignore

# 项目知识库入口文件
!AGENTS.md
!START-HERE-after-copy.md

# 项目知识库目录
!docs/
!docs/**
!tools/
!tools/**
EOF
```

---

## 4. 添加知识库文件

```bash
git add .gitignore AGENTS.md START-HERE-after-copy.md docs tools
```

---

## 5. 检查 Git 当前只管理目标文件

```bash
git status
```

确认里面只出现：

```text
.gitignore
AGENTS.md
START-HERE-after-copy.md
docs/...
tools/...
```

---

## 6. 提交

```bash
git commit -m "init project knowledge base"
```

---

## 后续日常使用

查看变更：

```bash
git status
```

提交知识库变更：

```bash
git add AGENTS.md START-HERE-after-copy.md docs
git commit -m "update project knowledge base"
```
