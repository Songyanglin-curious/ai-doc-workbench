# Logs

本目录记录已落地事实。

## 什么时候记录

- 只有当 AI 判断本次结果值得写日志，且用户已确认完成并允许维护时，才写 `logs/`
- `logs/` 是确认后的维护结果之一，不是“有变更就自动写”
- 如果只需要补索引或局部说明，不强制写 `logs/`

## 不记录什么

- 纯解释
- 临时分析
- 一次性查询
- 小修小问
- 未执行的想法
- 模糊讨论
- 原始需求全文

## 记录什么

- 本次改了什么
- 影响了哪些基线、路由或规则
- 跑了哪些验证
- 结果如何
- 是否需要后续跟进

## 使用方式

- 按年份创建子目录：`logs/YYYY/`
- 再按日期写文件：`MM-DD.md`
- 首次有日志时再创建年份目录，不预建空目录
