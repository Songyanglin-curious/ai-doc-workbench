# Rules

## Purpose

本文件承接 bootloader 以下的详细规则。

## 输入优先

- 当前用户输入优先于文档体系。
- 文档体系只用于补充上下文，不得替代用户当前任务目标。
- 当用户已明确给出文件、函数、报错、页面、样式问题或目录时，默认只读 `AGENTS.md` 和目标。
- 只有在证据不足以安全修改、无法判断边界、无法定位代码入口、任务明显扩散，或用户明确要求同步文档、做规划时，才进入 `docs/`。

## 搜索护栏

- 索引是按需定位工具，不是默认阅读链。
- 目标不明确时，先读 `docs/index.md`，再按需进入相关目录 `index.md` 或 `docs/context/map.md`。
- 先搜局部目录、模块目录、明确文件名或符号名，再扩大范围。
- 预计需要跨多个无关目录、命中约 30 个以上文件、或需要反复展开大文件时，先询问人。
- 若现有索引不足以支持定位，优先补索引，而不是直接扩大扫描范围。

## 短路径覆盖

- 明确目标时的默认路径：`AGENTS.md` + 用户指定目标。
- 不默认补读 `context/`、`baseline/`、`plans/`、`logs/`。
- 只有在以下情况才升级到更完整的阅读路径：
  - 输入不足以定位目标
  - 输入不足以判断边界、规则或代码入口
  - 任务明显跨模块或跨目录扩散
  - 当前目标与既有基线冲突
  - 用户明确要求同步文档、规划任务或复盘历史

## 复杂任务与计划

- 出现以下情况时，使用 `docs/plans/current.md`：
  - 跨模块
  - 跨会话
  - 改公共契约
  - 改权限或认证
  - 改数据结构
  - 改外部集成
  - 预计改动超过约 5 个文件
- 高风险任务在实现完成后，应做一次独立收尾复核。

## Ask First

- 需要新增 `docs/baseline/modules/<新模块>/`
- 需要大批量目录重组、重命名或迁移
- 需要越过既有搜索护栏
- 当前基线与实际实现冲突，且会改变长期结构判断
- 需要变更 `docs/baseline/principles.md` 或 `docs/baseline/core.md`
- 需要关闭高风险任务，且关闭判定依赖结构性判断

## 目录与边界

- `baseline/` 写当前认可的结构性判断，不写原始输入，不做资料汇总。
- `input/` 只放原始输入，不写项目结论。
- `references/` 只放外部协议、模板、规范和第三方说明，不承载项目主真相。
- `plans/` 只写复杂任务如何收口。
- `logs/` 只记已发生的落地事实。
- `docs/` 下每个目录都必须有 `index.md`。
- 模块目录使用中文业务名。

## 维护确认

- 维护确认不是任务结束后的固定流程。
- 小修、小问、临时分析、一次性查询默认不触发维护确认。
- 只有当 AI 判断这次结果值得写 `logs/`、更新索引或更新说明文档，且准备实际写这些维护内容时，才触发维护确认。
- 触发维护确认时，先确认两件事：
  - 本次任务是否已完成
  - 是否允许写入维护内容
- 用户说“还没完成”或“继续改”：
  - 继续任务
  - 不做维护
- 用户说“完成了，但先不维护”：
  - 结束本轮任务
  - 不写 `logs/`
  - 不改索引或说明文档
- 用户说“完成了，可以维护”：
  - AI 才可写 `logs/`
  - AI 才可更新索引、局部说明和相关导航
- 确认痕迹默认留在对话中，不额外写状态文本。
- 若维护目标触及长期基线、模块边界或结构方向，仍需人工确认，不默认改。

## AI 可在确认后维护

- `docs/input/index.md`
- `docs/references/index.md`
- 现有目录内的 `index.md` 导航更新，包括 `docs/index.md`、模块索引和其他已有索引
- `docs/context/index.md`
- `docs/context/map.md`
- 现有 `plans/` 中的任务收口记录
- `docs/logs/` 中已确认完成的常规事实记录
- `docs/baseline/modules/<已有模块>/index.md` 中的 `Code Map`、`Related Inputs`、`Related References`、`Change Notes` 与非结构性的 `Local Rules`
- 其他非结构性的局部说明

## AI 可提议但不默认改

- `docs/context/project.md`
- 已有模块的阶段性状态说明

## 必须先确认或强制人审

- `docs/context/rules.md`
- `docs/baseline/principles.md`
- `docs/baseline/core.md`
- `docs/baseline/modules/<已有模块>/index.md` 中的职责定义、边界定义、长期结构判断
- 新增 `docs/baseline/modules/<新模块>/`
- 大批量目录重组、重命名、迁移
- 任何会改变长期收敛方向的结构判断
- 高风险任务的关闭判定
