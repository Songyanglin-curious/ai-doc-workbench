# AGENTS.md

## 不确定时先问

当任务存在歧义，且继续执行会导致以下任一情况时，必须先问用户：

- 需要扩大到多个目录
- 需要启动子 Agent
- 需要全仓搜索
- 需要判断模块边界
- 需要猜测用户真正想改哪里
- 需要把“小任务”升级成“模块级调研”

AI 不得用“大范围搜索”代替澄清。

## 全局规则

- 使用中文思考和回答；必要时保留英文术语。
- 这是 AGE Lite bootloader：强骨架、按输入路由、弱内部结构。
- 顶层目录职责固定，目录内部默认从简，依靠 `index.md` 逐层组织。

## 输入优先

- 当前用户输入优先于文档体系。
- 文档体系只用于补充上下文，不得替代用户当前任务目标。
- 当用户已明确给出文件、函数、报错、页面、样式问题或目录时，默认不读 `docs/`，直接进入目标。
- 只有在证据不足以安全修改、无法判断边界、无法定位入口，或用户明确要求同步文档、做规划时，才进入 `docs/`。

## 维护确认

- 小修、小问、临时分析、一次性查询默认不触发维护确认。
- 只有当 AI 判断这次结果值得写 `logs/`、更新索引或更新说明文档，且准备实际写这些维护内容时，才触发维护确认。
- 触发后，先确认任务是否完成，以及是否允许写入维护内容。
- 未确认前，不默认写 `logs/`、索引或说明文档。
- 任务看起来完成后，AI 应先停下并汇报结果，提醒用户先审核实际效果。
- 如果本次结果可能需要写入 `logs/`、更新索引或维护说明文档，AI 可以在汇报后提示：用户确认结果无误后，可继续执行维护收口。
- 用户未明确确认前，AI 不得自动进入计划、日志、索引或说明文档维护。
- 用户确认任务完成并允许维护后，AI 才执行对应维护。

## 默认入口

1. 先读 `AGENTS.md`
2. 再按当前输入选择下一步

- 目标已明确：直接读用户指定目标
- 目标不明确，或需要在文档体系内继续定位：读 `docs/index.md`

## 明确目标时的短路径

- 默认路径：`AGENTS.md` + 用户指定目标
- 不主动补读 `docs/context/`、`docs/baseline/`、`docs/plans/`、`docs/logs/`

## 升级条件

- 用户输入不足以定位目标
- 用户输入不足以判断边界、规则或代码入口
- 任务明显跨模块或跨目录扩散
- 当前目标与既有基线冲突
- 用户明确要求同步文档、规划任务或复盘历史，也可以升级

## 最硬边界

- `input/` 不写结论
- `references/` 不承载项目主真相
- `plans/` 与 `logs/` 不定义系统本身
- `baseline/` 不退化为资料索引

## 详细规则入口

- 详细操作规则：`docs/context/rules.md`
- 基线优先级与适用边界：`docs/baseline/index.md`
