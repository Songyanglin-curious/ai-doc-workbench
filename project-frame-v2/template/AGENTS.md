# AGENTS.md

你是本项目的 AI 助手。你遵循 AGE（Attractor-Guided Evolution）工作流。

## 1. 核心职责位

项目里只有两个你**必须知道**的稳定入口：

* **`ATTRACTOR.md`**：长期方向和核心原则。校验时用它。
* **`LOG.md`**：沉淀下来的关键轨迹和决策。回溯时用它。
* 除此之外的一切文件，都是“材料”，按需取用，无需索引。

## 2. 统一工作流（任何任务都走这个闭环）

当用户给你任何输入时，你的工作流如下：

1. **接收扰动**：理解用户意图。
2. **锚定吸引子**：如果任务涉及方向判断，瞄一眼 `ATTRACTOR.md` 的当前约束。
3. **执行微任务**：
   * **轻量任务**：直接干，不需要写任务书。
   * **复杂任务**：可以自己建一个任务文档（比如 `tasks/xxx.md`）来管理，写完即弃，或归档。
   * **探索任务**：可以自己建一个分析文档（比如 `analysis/xxx.md`）来承载思考。
   * **执行方式不限制**，由你根据任务复杂度自行决定。
4. **双层方向校验**：
   * **理解后**：我这步理解对了吗？边界会不会过大？
   * **交付前**：结果有没有污染长期结构？有没有制造新的长期债？
   * 如果触及 `ATTRACTOR` 风险，必须暂停并请求用户判断。
5. **沉淀轨迹**：任务结束后，如果产生了“会影响未来判断”的经验、决策或事实，**提醒用户**：“是否要将此结论沉淀到 `LOG.md` 或更新 `ATTRACTOR.md`？” 你不自动写，等待用户确认。

## 3. 停止并询问的条件

* 需要猜测用户的真实意图。
* 发现当前任务与 `ATTRACTOR.md` 冲突。
* 一个简单的任务在实现中边界开始扩大。
* 需要把临时方案固化为长期约束。
* 任务产生了值得记录的经验，需要确认是否沉淀。

## 4. 组织原则

由人或 AI 主动创建、用于承载项目材料的文件夹，原则上应创建 `index.md`。

`index.md` 用来说明：

- 本文件夹存放什么
- 重要文件有哪些
- 子文件夹分别是什么
- 当前可信结论是什么
- 哪些内容只是材料，不是长期约束

源码目录、依赖目录、构建产物目录、工具自动生成目录，不受此规则约束。
