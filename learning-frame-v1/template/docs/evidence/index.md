# Evidence

`evidence/` 记录掌握证据，而不是学习流水账。

## 什么时候写

- 完成一次无 AI 测试
- 完成一次变式/迁移任务
- 发现明确夹生风险
- 一次验证失败暴露关键前置缺口
- 某能力准备进入 `baseline/`

## 什么时候不写

- 只是看了一段资料
- 只是让 AI 解释了一遍
- 只是做了跟随式练习
- 只是整理了笔记

## 证据记录模板

```text
日期：
对应能力节点：
验证方式：
是否使用 AI：否 / 学习阶段使用，验证阶段未使用
结果：pass / partial / fail
错误类型：概念不清 / 提取失败 / 迁移失败 / 执行粗心 / 前置缺口
证据摘要：
下一步：
是否建议更新 map：
是否建议写入 baseline：
```

## 评价规则

- `pass`：无 AI 条件下完成，并能解释关键边界。
- `partial`：能完成一部分，但依赖提示、卡在边界或迁移不稳。
- `fail`：不能完成，或发现前置能力不足。

失败不是坏结果；失败的价值是告诉 map 应该往哪里回退。
