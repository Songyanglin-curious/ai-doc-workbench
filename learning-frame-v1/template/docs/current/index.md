# Current

本目录只回答一个问题：

> 我现在正在补哪一个能力台阶？

不要把这里写成长期计划，也不要同时塞多个学习方向。当前台阶越小越好。

## 当前台阶卡片

```text
当前能力节点：
当前台阶：
为什么现在补它：
预计学习输入：
无 AI 验证方式：
回退条件：
下一步：
```

## 台阶大小标准

一个好台阶应该：

- 比当前能力多半步到一步
- 能在一次或几次学习中验证
- 有明确前置
- 有真实应用场景
- 验证失败时能知道回退到哪里

## 结束条件

当前台阶结束时只允许三种结论：

- `confirmed`：无 AI 验证通过，可以建议写 evidence，必要时更新 baseline
- `unstable`：能做一部分，但不稳，继续练或拆小
- `blocked-by-prerequisite`：发现前置缺口，回 map 修正
